## Key Design Decisions
1. **Skyvva-only data source** — No Salesforce EmailMessages, only Marketing Cloud via Skyvva
2. **Generic component** — Same LWC serves both Email Log and Kundenzufriedenheit via config
3. **Client-side pagination** — Skyvva returns all data in one JSON response
4. **No wrapper Apex** — STLG_SkyvvaCallout.invokeCalloutVariant1 already @AuraEnabled
5. **@wire with optionalFields** — Dynamic field loading for Account (Id) vs Case (AccountId)
6. **Page size default 5** — Configurable via design attribute
