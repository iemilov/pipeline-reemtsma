== Step 4: Implementation Approach ==

Mixed declarative + programmatic, following training pattern:

List/Card (follows training pattern exactly):
- pTA_WebinarsList LWC — uses getListUi with "PTA" ListView (mirrors pTA_TrainingsList)
- pTA_WebinarsCard LWC — tile component (mirrors pTA_TrainingsCard)
- Webinar__c "PTA" ListView — Published__c = true filter

Detail/Registration (new):
- pTA_WebinarDetail LWC — Knowledge content + registration
- pTA_WebinarRegistration LWC — form with full validation
- PTA_WebinarController Apex — registration, validation, detail queries
- PTA_WebinarControllerTest Apex — test coverage

Declarative:
- 2 new fields on Webinar__c (MaxParticipants__c, ParticipantCount__c)
- 1 new ListView on Webinar__c (PTA)
- 3 new Validation Rules on Webinar__c
- 9 new Custom Labels
- Permission set updates (DRKAReadOnlyExternal, DRKA_SystemAdministrator)
- Experience Cloud page configuration (new overview + updated detail)
- Flow deactivation (WebinarRegistrationFlow)
