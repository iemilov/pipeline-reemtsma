== Story Summary ==
Als Fachbereich / Sachbearbeiter in der Kundenverwaltung möchte ich, dass die Rückläufe
mittwochs nicht um 13:00 Uhr, sondern um 13:30 Uhr verarbeitet werden.

== Analysis ==
- Scheduler: STLG_CaseCreationScheduler (Schedulable class)
- Uses Custom Metadata STLG_2ndLvlCaseCreationConfig__mdt for config
- CMT record: Abo_Ruecklauf with ScheduledTime 11:00 UTC (13:00 CEST), ScheduledDays=3 (Wednesday)
- isSameTime() method uses ±5 min tolerance — no double processing risk with 30 min gap

== Implementation Approach ==
Declarative only — CMT metadata change + manual scheduling step.

1. Updated CMT Abo_Ruecklauf: ScheduledTime from 2025-08-01T11:00:00.000Z to 2025-08-01T11:30:00.000Z
2. Manual step: Schedule second job with cron '0 30 * * * ?'

== No Double Processing Guarantee ==
isSameTime compares hour (exact match) and minute (±5 tolerance).
- Config at 13:30: :00 job → |0-30|=30 > 5 → no match. :30 job → |30-30|=0 ≤ 5 → match.
- Config at 13:00: :00 job → |0-0|=0 ≤ 5 → match. :30 job → |30-0|=30 > 5 → no match.
