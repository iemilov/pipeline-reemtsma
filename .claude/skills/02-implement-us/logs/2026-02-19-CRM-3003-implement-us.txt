# Implementation Log: CRM-3003
# Date: 2026-02-19
# Story: TK: Bestandener TK triggert fälschlicherweise Kündigungsvorgang

## Step 1: Read User Story

Fetched CRM-3003 from Jira (Cloud ID: 2a9f60f6-99f9-4ab6-aedd-ea0fc09fe2d4).
Status: Already In Progress.

### Summary
Der Record-Triggered Flow STLGS_CalculateTestPurchaseStatus erstellt einen Kündigungsvorgang,
sobald der berechnete Ampelstatus eines TK-Cases "geschlossen" ist. Dabei wird nicht geprüft,
ob das TK-Ergebnis "nicht bestanden" war. Dadurch wird auch bei einem bestandenen TK ein
Kündigungsvorgang erstellt, wenn der Ampelstatus des Accounts bereits "geschlossen" war.

### Acceptance Criteria
- [ ] Bestandener TK bei Ampelstatus "geschlossen" erstellt keinen Kündigungsvorgang
- [ ] Nicht bestandener TK bei Ampelstatus "geschlossen" erstellt weiterhin Kündigungsvorgang (CRM-2820 bleibt erhalten)
- [ ] Fehlanfahrt und annullierte TK erstellen keinen Kündigungsvorgang

### Implementation Notes
Read from: implementation-design/CRM-3003/implementation-notes.md
- Root cause: Decision STLGS_Geschlossen only checks STLGS_TestPurchaseStatus__c == "geschlossen"
  but does not check STLGS_ResultTestPurchase__c == "nicht bestanden"
- CMT geschlossen_all returns "geschlossen" for both positive and negative results (correct behavior)
- Bug is in the flow, not in the Apex class or CMT

## Step 2: Explore Codebase Patterns

### Flow: STLGS_CalculateTestPurchaseStatus.flow-meta.xml
- Record-Triggered Flow on Case (RecordAfterSave, AsyncAfterCommit)
- Trigger conditions: RecordType=STLGS_TestPurchase, Status IsChanged, Type=Testkauf, Status=Testkaufergebnis geprüft
- Async path: GetAccount → Decision STLGS_Geschlossen → Get_Terminations → CreateTermination
- triggerOrder: 1500
- apiVersion: 61.0

### Bug Location: Decision STLGS_Geschlossen (lines 23-44)
Only one condition: $Record.STLGS_TestPurchaseStatus__c == "geschlossen"
Missing: $Record.STLGS_ResultTestPurchase__c == "nicht bestanden"

## Step 3: Implementation Approach

Declarative — Flow modification only. Minimal-invasive fix adding one condition to the existing
STLGS_Geschlossen decision node. No Apex changes needed. No new fields needed.
