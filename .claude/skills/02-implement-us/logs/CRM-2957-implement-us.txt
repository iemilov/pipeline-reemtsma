============================================================
CRM-2957 - VDE Gerät fest verbaut - Implementation Log
Epic: CRM-2453 (SD: VDE Prüfung durch Pro Service)
Date: 2026-02-02
============================================================

STORY SUMMARY
-------------
Pro Service needs to mark devices (Assets) as "permanently installed"
(Gerät fest verbaut). Permanently installed devices do NOT require VDE
inspection. When saving, a warning must remind the user to upload a
photo as proof via the Case comment.

IMPLEMENTATION APPROACH: Declarative (No Code)
----------------------------------------------
All requirements can be met with declarative Salesforce tools:
- Custom Field (Checkbox) for the "fest verbaut" flag
- Record-Triggered Flow (Before Save) for the warning message
- Validation Rule updates to bypass VDE checks for permanently installed devices

FILES CREATED
-------------
1. force-app/main/default/objects/Asset/fields/STLGS_PermanentlyInstalled__c.field-meta.xml
   - New Checkbox field "Gerät fest verbaut" on Asset
   - Default: false, Track History: true
   - Inline help text in German

2. force-app/main/default/flows/STLGS_WarnPermanentlyInstalled.flow-meta.xml
   - Record-Triggered Flow (Before Save) on Asset
   - Triggers when STLGS_PermanentlyInstalled__c changes to TRUE
   - Shows warning: "Achtung: Bitte laden Sie ein Foto als Nachweis
     über den Kommentar am zugehörigen Vorgang hoch, um die feste
     Verbauung des Geräts zu dokumentieren."

FILES MODIFIED
--------------
3. force-app/main/default/objects/Asset/validationRules/STLGS_ErrorFound.validationRule-meta.xml
   - Added condition: STLGS_PermanentlyInstalled__c = FALSE
   - Bypasses VDE error-found validation for permanently installed devices

4. force-app/main/default/objects/Asset/validationRules/STLGS_ErrorNOTFound.validationRule-meta.xml
   - Added condition: STLGS_PermanentlyInstalled__c = FALSE
   - Bypasses VDE inspection data validation for permanently installed devices

DEPLOYMENT COMMAND
------------------
sf project deploy start \
  --source-dir force-app/main/default/objects/Asset/fields/STLGS_PermanentlyInstalled__c.field-meta.xml \
  force-app/main/default/flows/STLGS_WarnPermanentlyInstalled.flow-meta.xml \
  force-app/main/default/objects/Asset/validationRules/STLGS_ErrorFound.validationRule-meta.xml \
  force-app/main/default/objects/Asset/validationRules/STLGS_ErrorNOTFound.validationRule-meta.xml \
  -o LottoUAT2

POST-DEPLOYMENT STEPS
---------------------
1. Add STLGS_PermanentlyInstalled__c field to the Asset page layout(s)
2. Add field-level security (FLS) for relevant profiles/permission sets
3. Verify the Flow is Active in Setup > Flows
4. Test: Mark an Asset as "fest verbaut" and confirm the warning appears
5. Test: Verify VDE validation rules are bypassed for permanently installed devices

ASSUMPTIONS
-----------
- The warning uses a customError (blocks save with message). If a non-blocking
  toast notification is preferred instead, this would need a different approach
  (e.g., LWC or a Before Save flow that does NOT block).
- The field is added to Asset but not yet to any page layout (manual step).
