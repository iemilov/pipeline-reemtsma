## Requirements
A Service Desk Case must not be closed when the checkbox "Vollstörung"
(STLGS_IsDisruption__c) is active. This means the store (Annahmestelle) is
currently not operational — no lottery ticket acceptance and no instant lottery
sales possible.

## Acceptance Criteria
- [x] When STLGS_IsDisruption__c = true and Case is set to "Closed", a
      validation error message appears
- [x] The error message is in German and clearly formulated
- [x] When STLGS_IsDisruption__c = false, the Case can be closed normally
- [x] When STLGS_IsDisruption__c = true but Case stays "Open", it is allowed
- [x] The validation rule does not throw errors on updates that are not closures

## Implementation Approach
**Declarative: Validation Rule** — This is purely a field-level data integrity
requirement that can be fully addressed with a validation rule. No Apex code
is needed.

### Rationale
- The requirement is a simple boolean + status check
- Validation rules are the standard Salesforce tool for preventing data changes
- Follows existing patterns in the codebase (27 existing validation rules on Case)
- Includes bypass switch for admin/data migration scenarios
- Scoped to STLGS_ServiceDeskCase record type as per SD context

## Files Created

### Validation Rule
- force-app/main/default/objects/Case/validationRules/STLGS_PreventClosureOnDisruption.validationRule-meta.xml

### Formula Logic
  AND(
    NOT($Setup.STLG_BypassSwitch__c.STLG_ValidationRuleSwitch__c),
    STLGS_IsDisruption__c = TRUE,
    IsClosed = TRUE,
    RecordType.DeveloperName = "STLGS_ServiceDeskCase"
  )

### Key Design Decisions
1. Uses IsClosed standard field (covers all closed statuses, not just "Closed")
2. Includes bypass switch ($Setup.STLG_BypassSwitch__c.STLG_ValidationRuleSwitch__c)
   consistent with all other validation rules
3. Scoped to RecordType.DeveloperName = "STLGS_ServiceDeskCase" (Service Desk cases only)
4. Error displayed on STLGS_IsDisruption__c field via errorDisplayField
5. Error message in German as specified in the story
