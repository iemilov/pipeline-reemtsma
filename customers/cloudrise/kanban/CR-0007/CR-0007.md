## User Story
As a logged-in user, I want a personalized dashboard that displays role-specific widgets and quick actions so that I can immediately see the information most relevant to my responsibilities without navigating through multiple pages.

The dashboard should adapt its layout and content based on the user's assigned role (e.g., Manager, Developer, QA) and allow basic customization of widget order and visibility.

## Acceptance Criteria
- [ ] Dashboard loads within 2 seconds and displays a greeting with the user's name
- [ ] Widgets are rendered based on the user's role configuration
- [ ] Users can reorder widgets via drag-and-drop
- [ ] Users can toggle widget visibility (show/hide) with preferences persisted across sessions
- [ ] A fallback default layout is shown for users without a specific role assignment
- [ ] Dashboard is fully responsive on tablet and desktop viewports
- [ ] Empty states are handled gracefully when a widget has no data to display

## Technical Notes
- Leverage the existing `@dnd-kit` library already used in the kanban board for widget reordering
- Store user widget preferences in the `app_settings` table keyed by user ID
- Widget configuration per role should be defined as a JSON mapping in a config file, not hardcoded in components
- Consider lazy-loading heavy widgets (charts, tables) to keep initial load fast
- Role data is available from the authenticated user context after login
